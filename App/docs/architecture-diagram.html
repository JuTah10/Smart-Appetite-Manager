🍲 Recipe Discovery Flow
1
User enters recipe query in RecipeDiscoveryPage (Web UI)
2
Request sent via SSE to SAM Gateway (:8000) with session ID
3
Orchestrator routes to RecipeAssistant (intent classifier)
4
RecipeAssistant classifies intent:
- INVENTORY_RECIPE_SEARCH -> RecipeInventorySearch
- GENERAL_RECIPE_SEARCH -> RecipeGeneralSearch
5
RecipeInventorySearch: calls get_ingredient_names from inventory DB, then complex_search / get_top_3_meals via Spoonacular API
6
Results scored by: pantry coverage (45%), preference match (25%), prep time (20%), missing count (10%)
7
Response with recipe_data JSON block parsed by frontend to render recipe cards
🛒 Shopping & Route Planning Flow
1
User requests deals in ShoppingPage (or clicks "Find Deals" from Inventory)
2
Can go via SAM Gateway (agent chat) or Inventory REST API (direct REST)
3
ShopperAgent calls find_deals_with_map -> queries Flipp API for each item (max 10 concurrent)
4
Filters non-grocery results, extracts weights via regex. Calls Overpass API to geocode stores
5
RoutePlannerAgent (optional): builds store-item matrix, generates 1/2/3-store combos + greedy route
6
Routes scored: price (35%), convenience (25%), coverage (25%), distance (15%) using Haversine + nearest-neighbor TSP
7
Response with shopper_map_data + route_plan_data JSON blocks rendered as interactive map + route cards
📦 Inventory Management Flow
1
User types "Add 2 kg rice, 1 L milk" in InventoryPage chat
2
InventoryOrchestrator delegates to IngredientParser
3
IngredientParser returns structured JSON with product_name, quantity, unit, category
4
InventoryOrchestrator presents confirmation table to user (MANDATORY - never skips)
5
User confirms -> InventoryDB executes insert_inventory_items with auto-merge duplicates
6
Written to inventory_manager.db (SQLite). Frontend refetches via REST API (:8001)
📷 Receipt Scanning Flow
1
User uploads receipt image via ScanReceiptDialog
2
Image sent to Inventory REST API (:8001) POST /api/receipt/scan
3
ReceiptScanner calls Vision LLM to parse receipt (store brand decoding, POS abbreviations)
4
enrich_product_codes: PLU lookup (500+ codes) -> Open Food Facts -> UPCitemdb
5
Items separated by is_food field. Food items presented for confirmation, non-food shown for reference
6
After user confirms, food items inserted into inventory_manager.db
